# Генерация объявлений Google Ads (RSA + ассеты) — HotWell.kz

Скилл-движок: `.claude/skills/generate-ads/generate_ads.py`
(валидация длин/редполитики + сборка через API). Полные правила:
`anatomy-of-a-good-ad.md` и `ad-assets-best-practices.md` (корень репозитория).

## Когда использовать
- Создать RSA-объявления для группы/кампании под конкретный ключ + лендинг.
- Дополнить кампанию ассетами (sitelinks, callouts, structured snippets).

## Бизнес-контекст HotWell.kz (использовать в текстах)
- **Продукт:** дома из СИП-панелей под ключ, домокомплекты; **своё производство**
  (завод в Алматы), монтаж своими бригадами. Работают с **2012 года**, по всему Казахстану.
- **Бренд:** HotWell.kz. Юр.лицо: ТОО «HOTWELL.KZ».
- **Каналы заявок:** WhatsApp и звонок (`+7 747 743 43 43`). Форм/почты нет — призывы
  ведут на «рассчитать», «написать в WhatsApp», «получить расчёт». Телефон в тексте
  объявления НЕ писать (только через ассет звонка/расширение).
- **Сильные стороны (правда, можно заявлять):** дом за ~30 дней; фундамент/кровля/доставка
  в цене; гарантия на дом 1 год; тёплый дом для зим РК; расчёт стоимости онлайн;
  цена и сроки в договоре; без скрытых доплат.
- **НЕЛЬЗЯ выдумывать:** рейтинги/отзывы с числами («4.9★ из 482»), «№1», «лучшие»,
  «дешевле всех» — без подтверждения. Не использовать бренды конкурентов (ModeX и т.п.).

### Реальные страницы для sitelinks (брать только существующие)
- `https://hotwellkz.kz/proekty.html` — каталог проектов (550+)
- Якоря лендинга: `#calc` (расчёт), `#includes` (что входит), `#works` (наши работы),
  `#tech` (технология СИП), `#reviews` (отзывы), `#contacts` (контакты), `#about` (о компании)

### Значения для structured snippets (реальные услуги/типы)
- Заголовок **Услуги**: СИП-дома, Дома под ключ, Бани из СИП, Дачные дома, Коттеджи, Гаражи, Фундаменты, Кровельные работы
- Заголовок **Типы**: Одноэтажные, Двухэтажные, С мансардой, Каркасные, Панельные, Под ключ

## Правила RSA (из anatomy-of-a-good-ad.md)
- **15 заголовков** (символы ≤30), из них **3 закреплены в позиции 1** — варианты ключа
  с гео (напр. «СИП-панельные дома Астана», «СИП дома в Астане», «Дома из СИП-панелей Астана»).
- Остальные **12 без закрепления**, покрывают ≥5 из 6 паттернов: оффер/УТП, доверие,
  срочность, гарантия, призыв, выгода/бренд.
- **4 описания** (≤90, целиться в 61–70). Углы: 1) услуга+скорость, 2) доверие+цена,
  3) отличие, 4) призыв. Каждое — законченная мысль с выгодой/призывом.
- **path1/path2** ≤15 символов (напр. `Астана` / `СИП-дома`).
- Закреплять ТОЛЬКО позицию 1 (одинарный пин). Описания и поз.2/3 — не закреплять.
- Если RSA несколько — каждое тестирует свой угол (скорость / доверие / цена), один и тот же лендинг.

## Правила ассетов (из ad-assets-best-practices.md)
- **Sitelinks:** 4–8 шт. title ≤25 (целиться 12–15), описания ≤35. Каждая на отдельную
  реальную страницу. Без общих «Узнать больше».
- **Callouts:** 8–12 шт. ≤25 символов, дифференцированные выгоды, не дублируют заголовки.
- **Structured snippets:** 2 заголовка (Услуги + Типы), 4–10 значений, ≤25 символов.
- **Business name** «HotWell.kz» и **логотип** уже добавлены в кампанию Астаны; для новых
  кампаний — добавлять отдельно (нужна проверка рекламодателя).

## Жёсткая редполитика (иначе отклонят)
Без «!» в заголовках (в описании максимум 1 на объявление); без двух КАПС-слов подряд
(одиночные «СИП», «РК» — можно); без эмодзи/«★★★»/«→»/повтора пунктуации; без телефонов
в тексте; без «№1/#1» и «лучший» без пруфа; без брендов конкурентов. (Полный список — §5–6 SOP.)

## Рабочий процесс
1. **Уточни вход** (если не задано): ключ/тема, ad_group_id и/или campaign_id, final_url
   (точный URL лендинга), path1/path2. ID групп/кампаний можно взять через
   `integrations/google-ads/ads_client.py` или `find_campaign_negatives.py`.
2. **Сгенерируй копию** сам (ты — конверсионный копирайтер) по правилам выше. Считай длину
   в СИМВОЛАХ (кириллица). Покрой 5–6 паттернов, не повторяйся.
3. **Запиши spec.json** (формат — в шапке `generate_ads.py`) во временную папку.
4. **Проверь:** `python .claude/skills/generate-ads/generate_ads.py validate spec.json`.
   Исправь все ✗ (ошибки длины/редполитики), пока не будет «ОШИБОК НЕТ».
5. **Покажи пользователю** итоговый список (заголовки/описания/ассеты) и дождись «ок».
   Ничего не публикуй без подтверждения.
6. **Собери (PAUSED):** загрузи креды (GOOGLE_ADS_* env) и
   `python .claude/skills/generate-ads/generate_ads.py build spec.json`.
   RSA создаются PAUSED; ассеты привязываются к кампании.
7. **Отчитайся:** что создано, и напомни, что всё на паузе до ручного включения.

## Заметки
- Всё создаётся на **PAUSED** — публикацию/включение делает человек.
- Аккаунт по умолчанию: `8802382037` (USD). Кампания Астаны: `23991449173`,
  её ad group: `206423446508`, лендинг `https://hotwellkz.kz/astana-sip-doma.html`.
- Дубли RSA Google отклоняет — если делаешь несколько, меняй набор заголовков/описаний.
