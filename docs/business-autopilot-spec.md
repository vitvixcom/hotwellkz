# Бизнес-автопилот: сеть агентов для создания и развития сайтов «под ключ»

> Цель: система, в которой агенты сами придумывают бизнес-идею, создают сайт,
> наполняют его, продвигают и анализируют — а человек (владелец) только
> одобряет ключевые решения через панель управления.
>
> Этот документ — **карта всего, что нужно решить и автоматизировать**, чтобы
> работало от А до Я. Это не код, а ТЗ/чек-лист для построения системы.

---

## 0. Что у нас уже есть (исходные инструменты)

Подключённые MCP-серверы определяют, что реально автоматизируется «из коробки»:

- **`wpwriter`** — полноценная автоматизация WordPress: создание сайта,
  настройка темы/меню/страниц, генерация контента и картинок, SEO, автоблог по
  расписанию, интеграция с Google Search Console, отслеживание позиций,
  WooCommerce (товары). Это **движок производства сайтов**.
- **`Gmail`** — отправка отчётов, уведомлений, запросов на подтверждение.
- **`github`** — хранение кода, версионирование, CI/CD для кастомных сайтов
  (не-WordPress) и для самой системы агентов.

Вывод: **самый быстрый и реальный путь — сайты на WordPress** (контентные,
нишевые, e-commerce). Кастомный код (Next.js и т.п.) — второй этап.

## 1. Типы бизнеса, которые система может создавать

Не обязательно новостной сайт. От типа зависит набор агентов и монетизация:

| Тип сайта | Монетизация | Сложность автоматизации | Инструмент |
|---|---|---|---|
| **Нишевый контентный / инфосайт** | Реклама (AdSense), партнёрки | Низкая ✅ | wpwriter автоблог |
| **Партнёрский (affiliate) обзорник** | Партнёрские комиссии | Низкая-средняя ✅ | wpwriter + товары |
| **Новостной / журнал** | Реклама, трафик | Средняя (нужна свежесть) | wpwriter автоблог |
| **E-commerce / дропшип** | Продажа товаров | Высокая (платежи, логистика) ⚠️ | wpwriter WooCommerce |
| **Сервис/инструмент (калькулятор, генератор)** | Подписка, реклама | Высокая (нужен код) ⚠️ | кастом + GitHub |
| **Локальные услуги (лидген)** | Продажа лидов | Средняя | wpwriter |

**Рекомендация для старта:** нишевый контентный + партнёрский. Полностью
цифровой цикл, минимум внешних зависимостей, всё автоматизируется уже сейчас.

## 2. Сеть агентов (роли)

Ниже — расширенный состав. Твои 5 агентов на месте, плюс те, без которых
«под ключ» не получится.

### 2.1. Оркестратор (главный «директор»)
- **Что делает:** управляет всем циклом, ставит задачи агентам, следит за
  состоянием, эскалирует тебе на подтверждение, запускается по расписанию.
- **Не агент-болтун, а диспетчер.** Решает: что сейчас делать с каждым сайтом.

### 2.2. Агент-Стратег (идеи)
- Генерирует бизнес-идеи: ниша, тип сайта, аудитория, способ монетизации.
- **Валидирует идею:** оценка спроса, конкуренции, потенциала трафика и дохода.
- Выдаёт «карточку идеи» с прогнозом → **ГЕЙТ 1: твоё одобрение**.
- Вопросы: откуда берёт данные о спросе (нужен источник keyword-данных).

### 2.3. Агент-Доменщик
- Подбирает свободные домены под одобренную идею, проверяет доступность,
  историю домена (не был ли в бане/спаме), цену.
- Готовит список вариантов → **ГЕЙТ 2: ты одобряешь покупку**.
- **Покупка домена = реальные деньги** → см. раздел 4 (платежи).

### 2.4. Агент-Строитель (реализация / код)
- **WordPress-путь:** создаёт сайт (`connect_site`/новый инстанс), ставит тему,
  настраивает структуру, меню, страницы, идентичность бренда, лого, иконку.
- **Кастомный путь:** генерирует код сайта, коммитит в GitHub, деплоит.
- Вопрос: где физически хостятся сайты (см. раздел 4 — хостинг).

### 2.5. Агент-Контентщик
- Контент-план (кластеры тем), генерация статей, картинок, мета-описаний,
  внутренняя перелинковка, категории/теги, расписание публикаций.
- Полностью покрыт `wpwriter` (autoblog, generate_content, generate_image).
- **ГЕЙТ 3 (опционально): выборочный QA качества** — важно для Google E-E-A-T.

### 2.6. Агент-SEO
- Технический SEO, ключевые слова, оптимизация заголовков/мета,
  schema-разметка, подключение Google Search Console, sitemap, скорость.
- Предлагает: что улучшить на основе позиций → отчёт/действия.

### 2.7. Агент-Аналитик
- Собирает метрики (трафик, позиции, доход), строит отчёты, выявляет
  что растёт/падает, даёт рекомендации оркестратору и тебе.
- Источники: Search Console (через wpwriter), позже — аналитика, AdSense.

### 2.8. Агент-Развития (роста)
- На основе аналитики решает: какие статьи обновить, какие темы добавить,
  где нарастить контент, стоит ли масштабировать или «убить» сайт.
- Это «мозг роста», замыкающий обратную связь.

### 2.9. Агент-Монетизации
- Ставит рекламные блоки/партнёрские ссылки, отслеживает доход, оптимизирует.

### 2.10. (Опц.) Агент-Хранитель/Безопасность
- Следит за здоровьем сайтов: битые ссылки, аптайм, бэкапы, обновления,
  взломы, спам в комментариях.

## 3. Жизненный цикл одного бизнеса (end-to-end)

```
[1] ИДЕЯ        Стратег генерирует + валидирует
                 → ГЕЙТ 1: ты одобряешь идею
                       │
[2] ДОМЕН       Доменщик ищет свободные домены
                 → ГЕЙТ 2: ты одобряешь покупку (деньги)
                       │
[3] ЗАПУСК      Строитель: создаёт сайт, тему, структуру, бренд
                       │
[4] КОНТЕНТ     Контентщик: план + первые N статей + картинки
                 → ГЕЙТ 3 (опц.): выборочный QA
                       │
[5] SEO         SEO-агент: Search Console, sitemap, оптимизация
                       │
[6] МОНЕТИЗАЦИЯ Агент-монетизации: реклама/партнёрки
                       │
[7] РОСТ (цикл) Аналитик собирает метрики →
                Агент-развития решает что улучшить →
                Контентщик/SEO выполняют → (повтор по расписанию)
                       │
[8] РЕШЕНИЕ     Масштабировать / поддерживать / закрыть сайт
                 → ГЕЙТ 4: крупные решения (новый бюджет, продажа сайта)
```

## 4. КЛЮЧЕВЫЕ ВОПРОСЫ, которые надо решить (без них «автомат» не взлетит)

Это самая важная часть. Технология готова — узкое место здесь.

### 4.1. Деньги и платежи 💳
- **Кто и как платит** за домены, хостинг, подписки, рекламу?
- Решение: отдельная карта/счёт с **лимитом бюджета на месяц**. Агент не
  тратит сам — формирует запрос, ты подтверждаешь в панели (ГЕЙТ).
- Нужен **бюджет-контроль**: лимит на сайт, общий лимит, стоп при превышении.

### 4.2. Домены
- Через какого регистратора? Нужен API (Namecheap, Cloudflare, GoDaddy и т.п.),
  чтобы агент мог проверять и (после одобрения) покупать программно.
- Решение по auto-renew, чтобы домены не терялись.

### 4.3. Хостинг / инфраструктура сайтов
- Где живут WordPress-сайты? Варианты:
  - Managed WP-хостинг с API (автосоздание инстансов) — идеально для автопилота.
  - Один сервер с мульти-WordPress.
  - WordPress.com / готовый провайдер, к которому цепляется wpwriter.
- Нужен способ **создавать новый сайт программно**, а не вручную.

### 4.4. Аккаунты и доверие (риск банов) ⚠️
- Google (Search Console, AdSense), партнёрки, соцсети **банят чисто
  автоматические аккаунты**. Нужна стратегия: реальные аккаунты, постепенный
  «прогрев», человеческий след, разные сайты — не одинаковые шаблоны.
- AdSense требует ручной модерации сайта перед одобрением.

### 4.5. Качество контента и E-E-A-T
- Google штрафует массовый ИИ-контент без ценности.
- Нужны: уникальность, проверка фактов, экспертность, авторы, выборочный QA.
- Решение: гейт качества + не «штамповать 500 статей в день».

### 4.6. Юридическое
- На кого регистрируются домены/доходы? ИП/самозанятость/компания.
- Налоги с дохода. Политики (Privacy Policy, GDPR/cookie) — генерируются авто.
- Авторские права на контент/картинки (ИИ-генерация — ок, но проверять).

### 4.7. Состояние и память системы
- **БД состояния**: каждый сайт = запись (идея, статус, домен, метрики, история
  действий, бюджет). Агенты читают/пишут БД, а не «помнят в чате».
- Без этого система не переживёт перезапуск и не масштабируется.

### 4.8. Оркестрация и расписание
- Планировщик (cron): раз в день/неделю будит оркестратор по каждому сайту.
- НЕ агенты крутятся 24/7 (сожжёт токены) — а дискретные запуски.

### 4.9. Гейты одобрения (Human-in-the-loop)
- Где именно нужен ты: идея, покупка (деньги), крупные решения, QA.
- Механизм: агент ставит задачу «на паузу» → панель/почта показывает запрос →
  ты жмёшь «одобрить/отклонить» → агент продолжает.

### 4.10. Обработка ошибок и «предохранители» 🛑
- Что делать при сбое (API упал, домен не купился, сайт лёг)?
- **Kill-switch**: общий стоп всей системы.
- Лимиты: макс. трат, макс. действий, алерты при аномалиях.
- Логи всех действий агентов (аудит: кто что сделал и почему).

### 4.11. Стоимость работы системы
- Токены ИИ (генерация), хостинг, домены, подписки.
- Считать **юнит-экономику**: сколько стоит запустить и вести 1 сайт vs доход.

### 4.12. Метрики и обратная связь
- Подключить Search Console (есть в wpwriter), позже — аналитику, доход.
- Без реальных метрик «агент-развития» работает вслепую.

## 5. Панель управления (как ты этим управляешь)

- **Дашборд:** список всех бизнесов/сайтов со статусами и метриками.
- **Очередь одобрений:** карточки «ждёт твоего решения» (идея/покупка/QA).
- **Кнопки действий:** «Придумай идею», «Реализовать», «Развить».
- **Доска агентов:** кто чем занят в реальном времени.
- **Бюджет:** текущие траты, лимиты, кнопка стоп.
- **Логи/история:** что агенты делали.
- Уведомления: дублируются на Gmail (отчёты + запросы на подтверждение).

Технически: веб-фронтенд (React/Next.js) + бэкенд (Node/Python) на
**Claude Agent SDK**, который запускает агентов и реализует гейты.

---

## 6. Технологический стек (предлагаемый)

- **Агенты:** Claude Agent SDK (TypeScript или Python).
- **Бэкенд/оркестратор:** Node.js или Python + планировщик (cron).
- **БД состояния:** Postgres (или SQLite на старте).
- **Панель:** Next.js + API к бэкенду.
- **Производство сайтов:** WordPress через `wpwriter` MCP.
- **Код кастомных сайтов:** GitHub MCP + CI/CD.
- **Уведомления:** Gmail MCP.
- **Внешние API:** регистратор доменов, хостинг, (позже) AdSense/аналитика.

## 7. Дорожная карта (этапы)

**Этап 0 — Фундамент**
- БД состояния, оркестратор-скелет, 1 агент (Стратег), простая панель,
  один гейт одобрения. Проверка цепочки «идея → одобрение».

**Этап 1 — MVP одного сайта (полуручной)**
- Стратег → ты выбираешь идею → (домен/хостинг пока вручную) →
  Строитель + Контентщик делают сайт на 15–20 статей → публикация.
- Цель: пройти весь цикл хотя бы с ручными вставками.

**Этап 2 — Автоматизация цикла**
- Планировщик, автоблог, SEO-агент, Search Console, отчёты на Gmail.
- Домены/платежи через API с гейтом одобрения.

**Этап 3 — Обратная связь и рост**
- Аналитик + Агент-развития: система сама приоритизирует доработки.

**Этап 4 — Портфель и масштаб**
- Один оркестратор ведёт N сайтов. Монетизация, юнит-экономика, kill-switch.

**Этап 5 — Кастомные сайты (не только WordPress)**
- Агент-Строитель генерирует код, GitHub, деплой — для сервисов/инструментов.

## 8. Решения (вход в Этап 0)

### ✅ Уже решено
1. **Тип первых сайтов:** контентный + партнёрский.
2. **Язык/рынок:** оба — RU и англоязычный (смешанный портфель для теста).
3. **Уровень контроля:** **автономно с лимитами** — агент запускает идеи сам в
   рамках бюджета и правил, владелец смотрит отчёты и вмешивается при алертах.

> ⚠️ Следствие выбора «автономно с лимитами»: критично важны **предохранители**
> — бюджет-лимиты, kill-switch, чёткие правила «что можно без спроса».
> Это поднимается в приоритет Этапа 0 наравне с ядром оркестрации.

### ❓ Ещё нужно решить (блокеры Этапа 2 — когда подключаем деньги/инфру)
4. **Регистратор доменов** (нужен API: Namecheap / Cloudflare / GoDaddy)?
5. **Хостинг:** где создаются WordPress-сайты программно?
6. **Бюджет:** месячный лимит (домены + хостинг + токены) и лимит на 1 сайт?
7. **Панель:** локально или на сервере (доступ отовсюду)?
8. **Юр.форма:** на кого домены/доход (ИП/самозанятость/компания)?

> Пункты 4–8 не блокируют Этап 0 (ядро + панель можно строить уже сейчас),
> но обязательны до того, как система начнёт тратить деньги и публиковать.
